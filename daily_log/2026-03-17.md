## Decisions and direction

- **SSA merges use block params, not phi nodes.** This decision was made in Phase 1 and today's if/else lowering proves it works at scale through branching control flow. Block params carry values into merge blocks cleanly.

- **Submain is the integration branch for frontend features.** Rather than merging each feature individually into main, the pattern emerging is to accumulate related work on submain and merge as a consolidated batch. Today's three-commit stack (structs → easy wins → semantic parity) shows this clearly.

- **Semantic parity is being built incrementally.** Phase 1 adds the semantic node types and analysis passes. The interpreter still runs off the raw AST in some paths — removing that AST path is a separate step that comes after parity is proven.

- **Validator grows alongside the lowering.** Phase 5 didn't just add lowering code — it added over 1,000 lines of validation. The pattern is that every new lowering capability ships with corresponding validation to catch malformed IR immediately.
