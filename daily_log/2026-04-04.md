## Decisions and direction

The test runner implementation made one notable architectural choice: rather than implementing `cx test` as a separate binary or entry point, the `--test` flag reuses the existing compiler pipeline end-to-end (parse, semantic, IR) and adds a test-specific execution phase after semantic analysis. The runtime's `call_semantic_func` was made public to support this. This is a clean design — the test runner sees exactly the same language the normal runner sees.
