## Decisions and direction

**Phase ordering jumped.** The backend roadmap lists Phase 12 (differential harness) and Phase 13 (Cranelift skeleton) as "Next" items that follow after Phase 11 surface area reduction. Today both were started in parallel with ongoing Phase 11 work. This is not a sequence violation — Phase 12 and 13 have no hard dependency on Phase 11 completion. The harness needs to exist before execution is attempted; the skeleton needs to exist before any Cranelift code runs. Starting them now while Phase 11 sub-packets continue to land is a pragmatic parallelization.

**Host boundary defined before first execution.** CX-24 establishes that JIT mode executes inside the host process (no subprocess fork), main's return value becomes the exit code, and the differential harness captures JIT output by running the Cx binary as a subprocess. This means the same harness infrastructure works for both interpreter and JIT paths — no in-process hooking required.

**Structured errors continue to improve.** CX-21 follows the pattern established by CX-19: every `unsupported!` macro call in the lowering pass is being replaced by named, context-rich `UnsupportedSemanticConstruct` errors. This improves both developer experience and downstream tooling.
