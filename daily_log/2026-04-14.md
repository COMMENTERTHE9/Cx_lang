## Decisions and direction

Three decisions visible in diffs today:

- **Overflow semantics now match the locked syntax.** The roadmap has said "wrapping at declared width" since the v4.3 lock, but until today arithmetic actually wrapped at `i128` and only truncated at assignment. `dc564a7` moves truncation into the Binary/Unary evaluator so the runtime finally matches the spec. This is the spec catching up to itself, not a new design decision.
- **Implicit return kept, even with optional semicolons.** `6c27da0` preserves the implicit-return distinction (`expr` at end of body = return; `expr;` at end of body = discard) by tagging semicolon presence per-statement in the parser. This commits the grammar to "trailing expression is meaningful syntax" rather than collapsing everything into statements. Supported by `t99_semi_suppresses_implicit_return.cx` being an `expected_fail`.
- **Audit triage tiers are real.** The audit report explicitly splits findings into must-fix / should-fix / known-limitation / defer-post-0.1, with rationale for each. That's a usable framework for 0.1 triage going forward — especially the "known limitation" class for things like tree-recursion depth that aren't bugs but will bite users.

No spec or syntax-decision docs changed today beyond the roadmap version bump. `docs/AGENT_OPERATING_DOCTRINE.md` from 2026-04-12 was not modified.
