## Decisions and direction

**IR is now the backend interface.** Phase 0.5 is a quiet but important architectural decision. Before today, the Backend trait took `&Program` (raw AST). Now it takes `&IrModule`. This means all backend work going forward assumes IR — there's no path back to the AST-based backend. The Cranelift and LLVM stubs were updated accordingly.

**Resolver is a real dependency graph, not a file loader.** The resolver doesn't just find and read files — it builds a `ResolvedProgram` with stable `ModuleId`s, import edges, and topological ordering. This is more infrastructure than strictly needed for "load another .cx file," but it's the right foundation for dead symbol elimination and circular import detection, both of which are roadmap items.

**t65/t66 flip to expected_fail on submain.** On main these tests pass because there's no resolver — the import block is parsed and validated but never resolved. On submain, the resolver correctly errors because the imported files don't exist. This is an intentional and correct state change: the tests were always going to fail at resolution time; now they do.

**`../` path traversal is a known v0.1 accepted risk.** The resolver documents that `../` paths can escape the project directory. This is tracked as a known limitation, not a bug.
