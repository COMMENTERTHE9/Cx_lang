## Landed today

- **`if` as an expression** (`0c03555`, tracker #046, `origin/submain`) — CONFIRMED. `if c { a } else { b }` now works as a value in assignment, return, implicit-return, and nested expression positions (e.g., `1 + if …`, `f(if …)`), mirroring `Expr::When`. The parser adds `if_expr` with a mandatory final `else` (the statement-form `Stmt::IfElse` is unchanged — else stays optional there). `else if` chains desugar into nested `Expr::If` nodes. Semantic analysis types the result from the then-branch (no strict branch unification yet — tagged as a coherent 0.3 follow-up). Eval reuses the existing #026 unknown-condition check. A single `unsupported!` line in `src/ir/lower.rs` ensures the JIT cleanly SKIPs if-expression fixtures rather than crashing. 245 insertions across 21 files (7 source files + 14 test fixtures). Commit gates: 243/0 unit, 418/0 total, 219/0 matrix, 137/82/0 parity. 11 new verification matrix fixtures including assignment, nested, return, missing-else error, unknown-condition error, and a statement-form no-else regression guard.

- **TBool wire-value matching in `when` patterns** (`67bcb01`, tracker #044, `origin/submain`) — CONFIRMED. A typed `b: bool = ?` stored the canonical wire value `TBool(2)`, but the `when` matcher only compared against `Value::Unknown` — so the `unknown` arm could never fire for typed booleans (silent wrong result). The fix makes bool literal patterns in `run_semantic_when` match both `Value::Bool(…)` / `Value::Unknown(_)` and their `TBool(0|1|2)` wire equivalents. Single-site change in `src/runtime/exec.rs` (33 lines changed). Four new test fixtures: typed unknown, arithmetic unknown (`&&` of two unknowns), untyped regression guard, typed definite over-match guard. Commit gates: 212/0 matrix.

First code sample fixed to use Cx syntax (not `let`), enums documented (declaration, `::` variants, function signatures since #019), `when`-as-expression and `_` exhaustiveness documented, `?` as unknown literal documented, string interpolation documented as shipped. "Deferred Post-0.1" reframed into two honest buckets (interpreter-only vs. not-implemented-anywhere). Two new runnable examples added (`examples/enums.cx`, `examples/when_match.cx`). ## Decisions and direction

**`if` as expression, Strategy A.** The commit message explicitly references a "ruling" choosing Strategy A — a real `Expr::If` AST variant with mandatory else in expression position, desugared `else if` chains, and no branch-type unification (deliberately deferred to 0.3 as a coherent follow-up alongside multi-statement value blocks in `when`). The design is conservative: a trailing `if`/`else` becomes the implicit-return value only when every branch ends in a trailing expression, keeping `return`/EarlyReturn working in classify-style patterns.

**TBool dual-representation is now treated as a system invariant.** The TBool fix (#044) and the earlier `if`-condition fix (#026) both address the same root pattern: two equivalent unknown representations coexist (`Value::Unknown` and `TBool(2)`), and every site that inspects unknowns must handle both. The commit explicitly frames this as a "JIT forward constraint" — when the backend gains TBool lowering, it must replicate the dual-representation match. This is effectively a language invariant now.

**README as design document.** The README re-sync (#041) reframed the "Deferred" section into honest buckets, surfacing what works in the interpreter but is not yet JIT-lowered vs. what is genuinely unimplemented. This is a meaningful project-direction signal: it acknowledges the interpreter is substantially further along than the JIT and documents the gap explicitly.

No spec or design document changes detected in `docment/`.
