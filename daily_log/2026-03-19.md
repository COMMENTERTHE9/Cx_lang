## Decisions and direction

**Array types are first-class in the semantic layer.** The addition of `SemanticType::Array(usize, Box<SemanticType>)` means array types now propagate through the full pipeline — declaration, assignment, function return, and type compatibility checks. Previously arrays produced `SemanticType::Unknown`, which meant type checking around arrays was effectively disabled. This fix was required for the last three matrix tests to pass.

**Generic return type substitution at call sites.** The `substitute_type_params` function resolves generic return types when a function is called, not when it is defined. This is the correct decision — it means `fnc identity<T>(x: T): T` called with an `i64` argument returns `SemanticType::I64`, not `SemanticType::TypeParam("T")`. Without this, downstream type checking of generic function results was broken.

**Handle operations are real, not stubs.** The runtime now performs actual arena insert/get/remove for handle operations, with stale detection producing `RuntimeError::StaleHandle`. This closes the gap between the semantic IR representation and runtime behavior for handles.

**Parser ordering matters for handle syntax.** `handle_drop` and `handle_val` had to be ordered before `method_call` in the parser combinator chain. This is a subtle precedence issue — both handle operations and method calls can start with an identifier followed by a dot-like syntax, and the parser was greedily consuming handle syntax as method calls.
