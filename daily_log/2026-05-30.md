## In progress / uncommitted work

All 8 example files in `examples/` have been substantially rewritten (280 insertions, 78 deletions across the working tree). This is the most significant uncommitted work in weeks and constitutes a full tutorial-style overhaul of the example suite:

- **hello.cx** — Expanded from 7 to 39 lines. Added block comment headers explaining variables, printing, string interpolation, and arithmetic. Renamed variables for tutorial clarity, added new arithmetic demonstrations.

- **fibonacci.cx** — Restructured from a simple `fib()` function call to a more pedagogical layout. Added a `classify()` helper demonstrating explicit `return` with semicolons. Replaced the recursive `fib()` call with an inline iterative while-loop, using `let`/assignment pairs to work within Cx's current declaration syntax. Added comments explaining function syntax, `fnc:` colon requirement, and implicit vs explicit return.

- **error_handling.cx** — Rewrote from a compact `safe_divide`/`compute` pair into a three-function chain (`divide`, `double_divide`, `triple_divide`) demonstrating `Result<T>`, `Ok()`, `Err()`, and the `?` try operator with progressive chaining. Each concept gets its own commented section.

- **fizzbuzz.cx** — Added block comment header and inline explanations for while-loop syntax and if/else chains. Spacing improved for readability. Logic unchanged.

- **arrays_and_loops.cx** — Restructured to teach array declaration syntax (`[SIZE: Type]`), for-loop ranges (`0..5`), and while-in array iteration with `*arr` dereference. Renamed `nums` to `scores`, added print labels.

- **generics.cx** — Added explanation of generic struct syntax and monomorphization. Changed from `ints`/`floats` to `p`/`names` (integer and string pairs) for clearer type parameter demonstration.

- **structs_and_methods.cx** — Major rewrite. Dropped `max_health`, `heal`, `show` methods; added `is_alive()` (returns bool) and `add_score()`. Added detailed comments explaining impl block syntax including the `alias: Type` pattern and the `impl_name` binding. Renamed `p` to `hero`.

- **tbool_uncertainty.cx** — Rewritten to teach three-valued logic more clearly. Removed the `when` block demonstrations (which may not match current semantics cleanly), replaced with `is_known()` on both bools and integers, and an unknown-propagation section showing `5 + ?` and `false && ?` short-circuit behavior.

This work is clearly pedagogical — turning terse, code-only examples into commented tutorial files that teach Cx concepts. It's not committed yet and sits on the `daily-log-2026-05-23` branch's working tree.

## Decisions and direction

The example rewrite represents a deliberate shift toward documentation and developer experience. Rather than adding new language features or closing tracker items, this work invests in making the existing language accessible to newcomers. The examples are being structured as a progressive tutorial: `hello.cx` → `fibonacci.cx` → `fizzbuzz.cx` → `arrays_and_loops.cx` → `structs_and_methods.cx` → `error_handling.cx` → `generics.cx` → `tbool_uncertainty.cx`.

Some noteworthy changes in the examples reflect practical constraints of the current language:
- `fibonacci.cx` moved from recursive `fib()` calls to iterative while-loops, possibly because the recursive version requires features that work better with the submain runtime fixes (BindingId indexing, lazy arena).
- `tbool_uncertainty.cx` dropped `when` block demonstrations, which may indicate awareness that `when` examples are better served by their own dedicated file or that the semantics around `when` + TBool need further stabilization.

The tracker #014 fix (token Display) is a small but user-facing quality improvement — every parse error in the compiler is now more readable. Combined with yesterday's batch (#010 lazy arena, #015 CLI flags, #030 CRLF fix), this brings the tracker sprint to 5 items closed in 2 days.
