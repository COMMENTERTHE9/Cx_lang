## Decisions and direction

The memory boundary audit forced two concrete decisions:

1. **StrRef cannot live in struct fields.** The semantic pass now rejects this at both field assignment and struct instantiation. This closes the last known escape path for arena-backed string references. The decision is consistent with the existing StrRef policy (cannot assign to variables, cannot return from functions) — struct fields were simply a gap in enforcement.

2. **Handle double-drop remains a silent no-op.** The audit confirmed that the generational index system makes double-drop safe (the second drop targets a slot whose generation has advanced), so it is documented as a known 0.1 limitation rather than a bug. A future warning is a post-0.1 consideration.

3. **Interpreter stack managed via dedicated thread.** Rather than reducing per-frame stack consumption (a larger optimization), the interpreter now spawns a 64 MB thread. This is sufficient for any reasonable recursive Cx program at 0.1. Per-frame reduction is a post-0.1 optimization tracked in the audit report.

The roadmap v5.0 update on submain represents a significant project milestone: all 9 hard blockers are now marked resolved with test evidence. The gap between this and a real 0.1 release is now dominated by integration (merging submain to main) and any remaining quality gate work, not by missing features or known soundness holes.
