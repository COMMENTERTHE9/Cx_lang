## Landed today

**Generics v1 — Phase 8 complete (0a97140, 8582d1a)**

The headline feature. Generic functions with single type parameter support shipped across six files: `ast.rs`, `parser.rs`, `semantic.rs`, `semantic_types.rs`, `lower.rs`, and `runtime.rs`. The implementation adds `Type::TypeParam` to the AST, `SemanticType::TypeParam` to the semantic layer, `type_params` fields on `FuncDef`/`SemanticFunction`/`FunctionInfo`, `<T>` syntax parsing, type parameter registration during analysis, T-consistency resolution at call sites, and TypeParam passthrough in `types_compatible`. Three verification matrix tests pass: t37 (basic generics), t38 (generic array returns with `[N]T`), and t39 (multi-argument generics).

This is 88 insertions across the frontend and backend — compact but touches every layer of the pipeline.

## Decisions and direction

- **Generics v1 scoped tightly**: Single type parameter only, no struct dependency, no bounds. This keeps it self-contained and lets v2 (multi-param) and v3 (bounds) layer on incrementally.
- **CI designed for cross-platform**: The `config.toml` fix shows awareness that the repo is primarily developed on Windows but CI runs on Ubuntu — platform-specific paths can't leak into config.
- **Branch guard exempts merges**: Merge commits into main are now allowed through CI without triggering the branch protection gate.
- **Backend IR stashed rather than force-merged**: Clean approach — stash the WIP, merge the stable features, come back to IR later.
