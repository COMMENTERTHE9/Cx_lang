## Status

Variables and assignment are stable in the interpreter. The JIT lowers typed and inferred declarations, reassignment, arithmetic over variables, and compound assignment. Some width-specific fixtures still SKIP.

## Typed Declaration

Declare a value by writing `name: Type = value`.

<SourceComment source="origin/submain:src/tests/verification_matrix/t122_vardecl_int_exit.cx" />
```cx
x: t64 = 42
assert_eq(x, 42)
y: t64 = 100
assert_eq(y, 100)
```

## Reassignment

`=` updates an existing binding.

<SourceComment source="origin/submain:src/tests/verification_matrix/t123_vardecl_reassign_exit.cx" />
```cx
x: t64 = 0
assert(x == 0)
x = 42
assert(x == 42)
x = 100
assert(x == 100)
```

## Local Inference

`let` creates a binding before the value is known. The assigned value determines the type.

<SourceComment source="origin/submain:src/tests/verification_matrix/t17_unknown_literal.cx" />
```cx
let x;
x = ?
print(x)
```

Local inference works. Do not document broad or global type inference as complete.

## Constants

`const` values are initialized at declaration and cannot be reassigned.

<SourceComment source="origin/submain:src/tests/verification_matrix/t56_const_basic.cx" />
```cx
const MAX_HP: t32 = 100
const GRAVITY: f64 = 9.8

print(MAX_HP)
print(GRAVITY)
```

## Compound Assignment

Use the standard compound operators: `+=`, `-=`, `*=`, `/=`, `%=`.

<SourceComment source="origin/submain:src/tests/verification_matrix/t151_var_compound_assign_exit.cx" />
```cx
x: t64 = 10
x += 5
assert_eq(x, 15)

x = 20
x -= 8
assert_eq(x, 12)

x = 6
x *= 7
assert_eq(x, 42)

x = 100
x /= 4
assert_eq(x, 25)

x = 17
x %= 5
assert_eq(x, 2)
```

The modulo form is `%=`.

## Unknown Values

`?` is executable in the interpreter. Unknown propagation is part of the current language surface. JIT support is incomplete.

<SourceComment source="origin/submain:src/tests/verification_matrix/t18_unknown_propagation.cx" />
```cx
let x;
x = ?
let y;
y = 5 + x
print(y)
let z;
z = false
let w;
w = z && x
print(w)
```
