## What Cx Is

Cx is a systems language for game engines. It has no garbage collector. It uses typed values, explicit movement, arenas, handles, and a small core.

Cx is early. Expect breaks.

The current version is `v0.1.0-rc`.

## Execution Status

The interpreter is the reference implementation.

The Cranelift JIT is Phase 15 active and parity-gated against the interpreter. It is incomplete. Current parity gate: 155 fixtures, 66 PASS, 89 SKIP, 0 PARITY_FAILs.

SKIP means a construct is not lowered to JIT code yet. PARITY_FAIL means the JIT disagrees with the interpreter. The current gate has no parity failures.

Source: [`docs/backend/cx_jit_parity_checklist.md`](https://github.com/COMMENTERTHE9/Cx_lang/blob/submain/docs/backend/cx_jit_parity_checklist.md).

## Stable In The Interpreter

- functions and direct calls
- forward and nested functions
- integer types `t8`, `t16`, `t32`, `t64`, `t128`
- `f64`
- `bool`, `str`, and `char`
- unknown values and `tbool`
- `if` / `else`
- `when`
- `while`, `for`, `loop`, `break`, `continue`
- fixed-size arrays
- structs, `impl`, methods, field read/write
- grouped enums
- generics for functions and structs
- `Result<T>`, `Ok`, `Err`, and `?`
- multi-file imports with `#![imports]`
- `#[test]`, `--test`, `assert`, `assert_eq`
- compound assignment with `+=`, `-=`, `*=`, `/=`, `%=`
- string interpolation
- `print`, `printn`, `read`, and `input`

## In Progress In The JIT

The JIT lowers integer arithmetic, comparisons, variable declarations, `if` / `else`, parts of loop control, direct calls, struct field read/write, arrays, compound assignment, logical `&&` / `||`, integer casts, `print` / `println` / `printn` for `t64`, and assertion built-ins.

The JIT does not yet lower `when`, method calls, `f64` operations, unary negation, enums, generics, handles, string arena operations, `Result<T>`, or `?`.

## Planned Or Thin

- `strref` exists as a boundary type. User-facing constructor syntax is not settled.
- C interop / FFI is post-0.1.
- The standard library is minimal core first.
- Broad type inference is not a finished feature. Local variable inference works.
