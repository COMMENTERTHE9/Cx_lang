## Current Frame

The interpreter is the reference implementation.

The Cranelift JIT is Phase 15 active and parity-gated against the interpreter. It is incomplete.

Current gate: 155 fixtures, 66 PASS, 89 SKIP, 0 PARITY_FAILs.

Source: [`docs/backend/cx_jit_parity_checklist.md`](https://github.com/COMMENTERTHE9/Cx_lang/blob/submain/docs/backend/cx_jit_parity_checklist.md).

## What PASS Means

PASS means the JIT result matches the stored expectation for that fixture.

Expected-fail fixtures pass when both paths reject the program. Output fixtures pass when stdout matches. Exit-code fixtures pass when assertions hold.

## What SKIP Means

SKIP means the construct is not lowered to JIT code yet. It exits cleanly with an unsupported-construct signal or a structured codegen error.

SKIP is not a parity failure.

## What PARITY_FAIL Means

PARITY_FAIL means the JIT and interpreter disagree. That count must stay zero.

## JIT-Lowered Today

- integer arithmetic and comparisons
- compound assignment with `+=`, `-=`, `*=`, `/=`, `%=`
- logical `&&` / `||`
- variable declarations, typed and inferred
- `if` / `else`
- `for` ranges
- parts of `while`, `loop`, `break`, and `continue`
- direct function calls
- struct literals and field read/write
- fixed-size array allocation, read, write, and function use
- `print`, `println`, `printn` for `t64`
- `assert` and `assert_eq`
- integer casts
- void returns and exit-code propagation

## Not Yet JIT-Lowered

- `when`
- method calls
- `f64` operations
- unary negation
- enums
- generics
- `Handle<T>`
- string arena operations
- `Result<T>` and `?`
- `read` and `input`
