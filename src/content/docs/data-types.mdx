## Status

The interpreter supports the 0.1 type surface. The JIT lowers integers, bool paths, arrays, structs, and part of `tbool`. Strings, `char`, `strref`, handles, generics, `Result<T>`, and `f64` operations are not complete in the JIT.

## Integers

Cx has signed integer types `t8`, `t16`, `t32`, `t64`, and `t128`.

<SourceComment source="origin/submain:src/tests/verification_matrix/t122_vardecl_int_exit.cx" />
```cx
x: t64 = 42
assert_eq(x, 42)
y: t64 = 100
assert_eq(y, 100)
```

Integer arithmetic and compound assignment are JIT-lowered, with some width coverage still marked SKIP.

## Floating Point

`f64` works in the interpreter. Float operations are SKIP in the JIT.

<SourceComment source="origin/submain:src/tests/verification_matrix/t55_f64_basic.cx" />
```cx
score: f64 = 3.14
velocity: f64 = 9.8
result: f64 = score + velocity
print(result)
```

## Bool And TBool

`bool` has `true` and `false`. Cx also supports unknown state through `?` and `tbool` behavior.

<SourceComment source="origin/submain:examples/tbool_uncertainty.cx" />
```cx
a: bool = true
b: bool = ?

print(is_known(a))
print(is_known(b))

let x;
x = true

when x {
    true    => print("is true"),
    false   => print("is false"),
    unknown => print("is unknown"),
}
```

Logical `&&` and `||` lower in the JIT. `when` over TBool does not.

## Strings

`str` works in the interpreter. String interpolation works in print calls. String arena operations are not JIT-lowered.

<SourceComment source="origin/submain:src/tests/verification_matrix/t75_string_interpolation.cx" />
```cx
name: str = "Cx"
version: t32 = 1
print("Language: {name}")
print("Version: {version}")
x: f64 = 3.14
print("Pi is about {x}")
```

## Char

`char` exists in the lexer, parser, semantic layer, and runtime. IR lowering rejects `Char`, so JIT support is planned.

<SourceComment source="origin/submain:src/tests/verification_matrix/t24_full_system_regression.cx" />
```cx
print('a' == 'b')
```

## Boundary Types

`strref` exists as a boundary type. User-facing constructor syntax is not settled.

`Container` exists around `copy_into(...)` behavior and related runtime plumbing.

## Handles

`Handle<T>` works in the interpreter. It is not JIT-lowered.

<SourceComment source="origin/submain:src/tests/verification_matrix/t24_full_system_regression.cx" />
```cx
let h;
h = Handle.new(99)
print(h.val)
h.drop()
```
