Two things happened today: parse errors got readable, and every example file in the repo was rewritten from scratch as a teaching tool.

## Human-readable parse errors

Tracker #014 closed with commit `6dfd06d` on submain. The change adds a `Display` impl for the `Token` enum with roughly 94 match arms, mapping every token variant to a name a developer would actually recognize. Keywords print as their source text (`let`, `fnc`, `when`), type keywords as their Cx names (`t64`, `bool`, `void`), literals as descriptive categories like `"string literal"` or `"integer literal"`, and operators as their actual symbols (`+`, `==`, `->`).

The error formatter in `main.rs` switched from `{:?}` (Rust debug formatting) to `{}` (Display), so a parse error that used to say something like:

```
ExpectedFound { expected: ['PunctColon', ...], found: ... }
```

now says:

```
found 't64' expected ':'
```

The implementation is localized: 97 new lines in `lexer.rs`, one format specifier change in `main.rs`. No test fixtures needed updating because none were asserting on the old debug wording. Small change, real usability improvement for anyone hitting a syntax error.

## Example suite overhaul

All 8 files in `examples/` were substantially rewritten (280 insertions, 78 deletions). This is uncommitted work sitting on the working tree, but it represents the most significant documentation effort in weeks. The examples shifted from terse, code-only snippets to tutorial-style files with block comments explaining each concept.

A few of the more interesting rewrites:

**hello.cx** went from 7 lines to 39. It now walks through variables, printing, string interpolation, and arithmetic with commented sections for each.

**fibonacci.cx** moved from a recursive `fib()` call to an iterative while-loop approach. This is a practical choice given Cx's current declaration syntax. The file also picks up a `classify()` helper demonstrating explicit `return` with semicolons, and comments explaining the `fnc:` colon requirement and implicit vs explicit return.

**error_handling.cx** was rebuilt around a three-function chain (`divide`, `double_divide`, `triple_divide`) that progressively demonstrates `Result<T>`, `Ok()`, `Err()`, and the `?` try operator. Each concept gets its own section.

**structs_and_methods.cx** saw the biggest structural change. It dropped `max_health`, `heal`, and `show` methods in favor of `is_alive()` (returns bool) and `add_score()`, with detailed comments covering impl block syntax including the `alias: Type` pattern and the `impl_name` binding.

**tbool_uncertainty.cx** dropped its `when` block demonstrations entirely, replacing them with `is_known()` on both bools and integers, and an unknown-propagation section showing `5 + ?` and `false && ?` short-circuit behavior. The `when` removal likely reflects awareness that those examples need their own dedicated file or that `when` + TBool semantics need further stabilization.

The examples are being structured as a progressive tutorial sequence: `hello.cx` through `fibonacci.cx`, `fizzbuzz.cx`, `arrays_and_loops.cx`, `structs_and_methods.cx`, `error_handling.cx`, `generics.cx`, and finally `tbool_uncertainty.cx`.
