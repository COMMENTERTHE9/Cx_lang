## The merge gap

Submain is 21 commits ahead of main. It has been for a while now, and the gap is the single largest structural concern in the project at this point.

Those 21 commits are not minor. They include Phases 10 and 11 of the IR backend (while/for/loop/break/continue lowering, unary expression lowering), the error model (`Result<T>`), integer overflow enforcement, optional semicolons everywhere, two full audits (parser/semantic/interpreter agreement, memory boundary soundness), a diagnostics readability pass, and the roadmap bumped to v5.0 with all 9 hard blockers checked off.

Meanwhile, main sits at v4.8 with 8 hard blockers still unchecked. The test gap tells the same story: 78/78 on main versus 117/117 on submain. Merging submain would close a 39-test gap in a single action.

Every daily log for the past week has listed "merge submain into main" as the top predicted next step. It has not happened yet. That is not a crisis, but it is worth being honest about: the longer submain diverges, the harder the eventual merge becomes if new work starts landing on either branch independently.

## What's next

The priority list has not changed, and that is itself informative. Ranked by leverage:

1. **Merge submain to main.** This is the single highest-impact action available. It would bring main from v4.8 to v5.0, check off all remaining hard blockers, and close the 39-test gap between branches.
2. **Continue Phase 11 expression lowering.** The remaining unsupported expression kinds in `lower.rs` (ArrayLit, Index, MethodCall, StructLit, HandleVal, HandleDrop, Range) are the natural next backend targets.
3. **Fix the IR backend test failure.** Submain's Rust test suite reports 123 passed and 1 failed. Clearing that single failure would bring the suite to fully green.
4. **Clean up stale daily-log branches.** 24 unmerged branches on the remote is not blocking anything, but it is growing.

Quiet days are part of development. The project is not stuck in any deep sense. But the submain merge has been deferred long enough that it is worth flagging as the clear next move whenever work resumes.
