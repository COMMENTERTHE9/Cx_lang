Today landed as a deliberate reset: a large feature set adding arrays, index writes, while-in loops, and struct scaffolding was merged and then immediately reverted. The revert is the real signal. We pushed on multiple language surfaces at once and confirmed the system did not hold together reliably enough to keep those changes on `site`-tracked development.

The important detail is scope. This was not an isolated parser tweak. The change touched AST/frontend behavior, semantic analysis, IR lowering, and runtime paths. Once a feature spans that much of the compiler/runtime boundary, regressions compound quickly, and debugging cost rises faster than feature velocity.

Design-wise, this validates a stricter staging strategy for compound types and indexed mutation. Arrays and struct scaffolding create structural invariants that must be consistent from parse tree shape through lowering contracts and runtime layout assumptions. If any layer lags, the entire stack becomes ambiguous and test outcomes stop being trustworthy.

Next step is to re-land in narrower slices: parser + AST shape first, then semantic checks, then lowering, then runtime support, each with targeted tests and no silent cross-layer assumptions. The goal is still the same feature set, but with integration points proven incrementally instead of betting on one big pass.
