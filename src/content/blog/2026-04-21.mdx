## Roadmap stall

The roadmap on main (v4.8) lists several unchecked hard blockers for 0.1:

- Basic test runner
- Minimal error model
- Integer overflow behavior enforced
- Semicolons enforced consistently
- Parser/semantic/interpreter agreement on all constructs
- No known soundness holes in memory boundary model
- All examples pass

- Roadmap and spec match actual behavior

Several of those items already have completed or partially completed work on submain. The test runner exists there. The error model exists there. Integer overflow enforcement exists there. Semicolons are handled there. Diagnostics were addressed there. But because none of it has reached main, the roadmap items stay unchecked. The submain roadmap (v4.7) has some of these marked off, reflecting the real state of the work, but the two roadmap versions have diverged.
