## Unary lowering, documented

Cx's IR has no dedicated negate or boolean-not instructions. Instead, both unary operators are lowered to two-operand forms:

- `Op::Minus` becomes `0 - value`
- `Op::Not` becomes `value == 0`

This means backends only need to handle one instruction shape for arithmetic and comparison, rather than carrying separate unary variants. It is a deliberate encoding choice, but it is not obvious from reading the IR code alone. The CX-6 commit (`2d665a4`) adds 25 lines of comments explaining the rationale directly in `lower.rs`.

The commit is branch-local and has not been merged to main or submitted as a PR yet. It sits on top of the `stokowski/cx-6-document-unary-lowering` branch, which inherits 21 older commits from the submain lineage (Phases 10-11 IR backend work, audits, error model). The documentation commit itself carries zero integration risk since it is comments-only.

## The submain question

Submain divergence remains the dominant structural issue in the project. The submain lineage carries 22 commits not on main, spanning about 35 days. Main is still at v4.9 with 78 tests; submain carries v5.0 with 117 tests and all hard blockers resolved. New feature branches (like the CX-6 documentation branch) are forking from submain rather than main, which means the main branch roadmap and the actual state of the language continue to drift apart.

This is not new information. It has been noted in multiple daily logs. But it is worth repeating because it affects how every other piece of work lands. Until submain is merged, main does not reflect where Cx actually is.
